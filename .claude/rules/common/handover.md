# Handover Document Rule

## Mandatory Update

**每次修改项目文件时，必须同步更新 `HANDOVER.md`。**

### 触发条件

以下操作必须更新交接文档：
- 创建新文件
- 修改现有文件（实质性变更）
- 删除文件
- 完成一个任务或里程碑
- 做出技术决策
- 遇到并解决问题

### 更新内容

更新 `HANDOVER.md` 中的以下部分：

1. **最后更新日期** - 文件顶部
2. **当前阶段** - 如阶段变化则更新
3. **进度表** - 更新对应阶段的状态
4. **已完成的工作** - 在表格中添加新行
5. **文件清单** - 如有新文件则更新
6. **技术决策记录** - 如有新决策则添加
7. **下一步工作** - 根据当前进度更新
8. **变更日志** - 在末尾添加新行

### 变更日志格式

```markdown
| 日期 | 操作 | 文件 | 说明 |
|------|------|------|------|
| YYYY-MM-DD | CREATE/UPDATE/DELETE | path/to/file | 简要说明 |
```

### 状态标记

- ⬜ 未开始
- 🔄 进行中
- ✅ 完成
- ❌ 阻塞
- ⏸️ 暂停

### 目的

确保每次交接给下一个 agent 时，可以：
- 快速了解项目当前状态
- 知道哪些工作已完成
- 知道下一步该做什么
- 了解之前的技术决策和原因
