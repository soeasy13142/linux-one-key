**Source PRD**: `.claude/prds/linux-security-hardening.prd.md` (§2.4, §2.6, §2.7)

## Summary

为 linux-one-key 项目添加三个新安全模块：用户管理（创建用户/密码/SSH密钥/sudo）、内核安全加固（sysctl参数/内核模块限制）、文件系统安全（目录权限/SUID审计/无主文件检查）。每个模块遵循现有模式：wizard函数 + i18n + 单元测试 + 菜单集成。

## Tasks

### Task 1: 用户管理模块 (`scripts/security/users.sh`)

- **Action**: 创建用户管理模块，包含以下功能：
  - `create_admin_user()` — 交互式创建用户，添加到 sudo(wheel) 组
  - `set_user_password()` — 设置用户密码（带基本强度检查：≥8位）
  - `setup_user_ssh_key()` — 为用户生成 Ed25519 密钥对并配置 authorized_keys
  - `configure_sudo_nopasswd()` — 可选配置 sudo NOPASSWD
  - `run_users_wizard()` — 完整用户管理向导（创建→密码→SSH密钥→sudo）
  - `check_sudo_group()` — 检测 sudo/wheel 组名（Ubuntu=sudo, CentOS=wheel）
  - `validate_username()` — 验证用户名合法性（字母开头，仅含字母数字下划线，3-32字符）
- **Mirror**: `scripts/security/audit.sh` 的模块结构（guard → constants → internal → public → wizard）
- **OS detection**: `case "${DETECTED_OS}"` 用于选择 sudo 组名
- **确认流程** (每步 `confirm()` 后才执行):
  ```
  run_users_wizard():
    1. 显示向导说明 → confirm("是否开始用户管理配置？")
    2. 提示输入用户名 → validate_username → confirm("将创建用户 xxx 并添加到 sudo 组，确认？")
    3. 设置密码 → confirm("是否为 xxx 设置密码？")
    4. 生成 SSH 密钥 → confirm("是否为 xxx 生成 Ed25519 SSH 密钥？") → 显示密钥路径
    5. sudo 免密 → confirm("是否配置 xxx 的 sudo NOPASSWD？") → 显示安全风险警告
    6. 显示配置摘要
  ```
- **Validate**: `shellcheck -x scripts/security/users.sh && bats tests/unit/users.bats`

### Task 2: 内核加固模块 (`scripts/security/kernel.sh`)

- **Action**: 创建内核加固模块，包含以下功能：
  - `apply_sysctl_params()` — 生成 `/etc/sysctl.d/99-hardening.conf` 并应用
  - `disable_kernel_modules()` — 禁用不需要的内核模块（cramfs, freevxfs, hfs, hfsplus, udf, usb-storage）
  - `restore_sysctl_backup()` — 回滚 sysctl 配置
  - `run_kernel_wizard()` — 完整内核加固向导（sysctl → 模块限制 → 验证）
  - sysctl 参数（来自 PRD §2.6.1）：tcp_syncookies, rp_filter, accept_redirects=0, send_redirects=0, accept_source_route=0, log_martians=1, icmp_echo_ignore_broadcasts=1, ip_forward=0, randomize_va_space=2, suid_dumpable=0
- **Mirror**: `scripts/security/audit.sh` 的备份-应用-验证模式
- **Config**: 使用 `config/sysctl/hardening.conf` 模板，脚本中 `sed` 替换或直接 `cat` 生成
- **确认流程** (每步 `confirm()` 后才执行):
  ```
  run_kernel_wizard():
    1. 显示向导说明 → confirm("是否开始内核安全加固？")
    2. 列出将要设置的 sysctl 参数概要 → confirm("是否应用以上 sysctl 安全参数？")
       - 备份现有 /etc/sysctl.d/ → 生成新配置 → sysctl --system
    3. 列出将要禁用的内核模块 → confirm("是否禁用以上内核模块？")
       - 某些模块禁用失败时 log_warn 跳过，不阻断
    4. 显示配置摘要 + sysctl 验证结果
  ```
- **Validate**: `shellcheck -x scripts/security/kernel.sh && bats tests/unit/kernel.bats`

### Task 3: 文件系统安全模块 (`scripts/security/filesystem.sh`)

- **Action**: 创建文件系统安全模块，包含以下功能：
  - `check_critical_permissions()` — 检查关键目录权限（/etc/passwd=644, /etc/shadow=640, /etc/ssh/sshd_config=600, /root=700, /tmp=1777）
  - `audit_suid_sgid()` — 扫描并列出所有 SUID/SGID 文件，标记可疑项（仅报告，不修改）
  - `check_orphan_files()` — 查找没有属主的文件
  - `fix_critical_permissions()` — 交互式修复权限不正确的关键文件
  - `run_filesystem_wizard()` — 完整文件系统安全向导（权限检查→SUID审计→无主文件→修复建议）
- **Mirror**: `scripts/security/audit.sh` 的审计-报告模式
- **SUID handling**: 仅报告 + 建议，不自动修改。列出标准 SUID 文件（passwd, sudo, su 等）和可疑文件
- **确认流程** (每步 `confirm()` 后才执行):
  ```
  run_filesystem_wizard():
    1. 显示向导说明 → confirm("是否开始文件系统安全检查？")
    2. 检查关键目录权限 → 显示结果
       - 如果有问题项 → confirm("发现 N 个权限问题，是否逐一修复？")
         → 逐个显示问题文件 + 当前权限 + 建议权限 → confirm("是否修复 xxx？")
    3. SUID/SGID 审计 → 显示扫描结果（仅报告，无需确认修改）
       → confirm("是否继续检查无主文件？")
    4. 无主文件检查 → 显示结果（仅报告）
    5. 显示安全检查摘要
  ```
- **Validate**: `shellcheck -x scripts/security/filesystem.sh && bats tests/unit/filesystem.bats`

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| sysctl 参数在某些 VPS 内核中不可用 | MEDIUM | `sysctl -w` 失败时 `|| true` + 日志警告，不阻断流程 |
| 内核模块在容器/云环境中无法禁用 | HIGH | `modprobe -r` 失败时跳过并记录，不影响其他步骤 |
| sudo 组名在不同发行版不一致 | LOW | 通过 `detect.sh` 已有的 `DETECTED_OS` 自动选择 sudo/wheel |
| SUID 审计列出大量文件影响体验 | MEDIUM | 过滤标准 SUID 文件（/usr/bin/passwd 等），仅显示非标准项 |
| 用户创建后 SSH 密钥配置涉及目录权限 | MEDIUM | 严格设置 .ssh=700, authorized_keys=600，并验证 |
