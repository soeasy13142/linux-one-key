**项目名称**: linux-one-key
**版本**: v1.0
**日期**: 2026-06-20
**作者**: charliepan

---

## 1. 项目概述

### 1.1 背景

新购买的云服务器（如阿里云、腾讯云、AWS、DigitalOcean 等）默认配置存在诸多安全隐患：
- 默认开放 22 端口，root 用户可直接密码登录
- 未配置防火墙，所有端口暴露
- 无入侵检测和防护机制
- 系统未及时更新安全补丁

### 1.2 目标

提供一个**交互式一键脚本**，通过 SSH 连接到新服务器后运行，快速完成安全加固配置。

### 1.3 核心原则

| 原则 | 说明 |
|------|------|
| **安全优先** | 所有配置默认采用最安全选项 |
| **交互确认** | 每个关键步骤前显示说明并要求用户确认 |
| **可回滚** | 每次修改前备份原始配置，出错可恢复 |
| **幂等性** | 重复运行不会产生错误或重复配置 |
| **兼容性** | 支持 CentOS 7+/8/9, Ubuntu 20.04/22.04/24.04, Debian 11/12 |

## 2. 功能需求

### 2.1 SSH 安全加固

#### 2.1.1 修改 SSH 端口

| 项目 | 说明 |
|------|------|
| **功能** | 修改 SSH 默认端口（22 → 自定义端口） |
| **交互** | 提示用户输入新端口号，默认 2222，验证端口未被占用 |
| **配置文件** | `/etc/ssh/sshd_config` |
| **关键参数** | `Port <新端口>` |
| **备份** | 修改前备份 `sshd_config.bak.日期时间` |
| **验证** | 修改后自动检测新端口是否监听 |
| **回滚** | 提供一键恢复默认端口 |

#### 2.1.2 生成 SSH 密钥对

| 项目 | 说明 |
|------|------|
| **功能** | 在服务器上为指定用户生成 Ed25519 密钥对 |
| **密钥类型** | Ed25519（更安全、更短） |
| **交互** | 提示输入密钥保存路径和密码短语（passphrase） |
| **输出路径** | `~/.ssh/id_ed25519` 和 `~/.ssh/id_ed25519.pub` |
| **authorized_keys** | 自动将公钥添加到 `~/.ssh/authorized_keys` |
| **权限设置** | `.ssh` 目录 700，`authorized_keys` 600 |

#### 2.1.3 禁止 root 远程登录

| 项目 | 说明 |
|------|------|
| **功能** | 禁止 root 用户通过 SSH 登录 |
| **配置参数** | `PermitRootLogin no` |
| **前置检查** | 确保已创建具有 sudo 权限的普通用户 |
| **交互** | 如果没有普通用户，提示创建一个 |
| **风险提示** | 明确告知用户此操作后 root 将无法 SSH 登录 |

#### 2.1.4 禁止密码登录 & 强制密钥登录

| 项目 | 说明 |
|------|------|
| **功能** | 禁用密码认证，仅允许密钥认证 |
| **配置参数** | `PasswordAuthentication no`、`PubkeyAuthentication yes`、`ChallengeResponseAuthentication no` |
| **前置条件** | 确保用户已正确配置 SSH 密钥 |
| **交互** | 检测 `authorized_keys` 是否有有效密钥，无则警告 |
| **顺序** | 必须在密钥配置完成后再执行此步骤 |

#### 2.1.5 其他 SSH 安全配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| `MaxAuthTries` | 3 | 最大认证尝试次数 |
| `LoginGraceTime` | 60 | 登录超时时间（秒） |
| `ClientAliveInterval` | 300 | 客户端心跳间隔 |
| `ClientAliveCountMax` | 2 | 最大心跳丢失次数 |
| `AllowUsers` | 指定用户 | 限制可登录用户 |
| `X11Forwarding` | no | 禁用 X11 转发 |
| `MaxSessions` | 2 | 最大会话数 |

### 2.2 防火墙配置

#### 2.2.1 安装防火墙

| 系统 | 防火墙工具 |
|------|------------|
| Ubuntu/Debian | UFW (Uncomplicated Firewall) |
| CentOS 7 | firewalld |
| CentOS 8+/9 | firewalld 或 nftables |

#### 2.2.2 基础规则配置

| 规则 | 说明 |
|------|------|
| **默认策略** | 拒绝所有入站，允许所有出站 |
| **SSH 端口** | 开放用户设置的 SSH 端口（非默认22） |
| **HTTP/HTTPS** | 可选开放 80/443（Web 服务器场景） |
| **ICMP** | 可选允许 ping（用于监控） |
| **其他端口** | 交互式询问用户需要开放的端口 |

#### 2.2.3 交互流程

```
1. 检测系统类型，选择防火墙工具
2. 显示当前防火墙状态
3. 询问需要开放的端口（SSH 端口自动添加）
4. 询问是否需要 HTTP/HTTPS
5. 询问是否允许 ping
6. 应用规则并显示最终配置
7. 验证规则生效
```

#### 2.2.4 防火墙管理命令提示

脚本完成后显示常用管理命令：
- 添加/删除规则
- 查看状态
- 临时禁用/启用

### 2.3 Fail2Ban 入侵防护

#### 2.3.1 安装配置

| 项目 | 说明 |
|------|------|
| **安装** | 自动检测包管理器安装 fail2ban |
| **服务** | 启动并设置开机自启 |
| **配置目录** | `/etc/fail2ban/` |

#### 2.3.2 SSH 防护配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `bantime` | 3600 | 封禁时间（秒），默认 1 小时 |
| `findtime` | 600 | 检测时间窗口（秒） |
| `maxretry` | 5 | 最大失败次数 |
| `banaction` | firewallcmd-ipset | 封禁动作（与防火墙集成） |

#### 2.3.3 自定义 Jail 配置

```ini
[sshd]
enabled = true
port = <用户设置的SSH端口>
filter = sshd
logpath = /var/log/auth.log    # Debian/Ubuntu
logpath = /var/log/secure      # CentOS/RHEL
maxretry = 5
bantime = 3600
```

#### 2.3.4 可选扩展防护

| 防护类型 | 说明 |
|----------|------|
| nginx-http-auth | Nginx HTTP 认证防护 |
| postfix-sasl | Postfix SMTP 认证防护 |
| recidive | 重复封禁（多次被封的 IP 延长封禁） |

#### 2.6.1 Sysctl 安全参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `net.ipv4.tcp_syncookies` | 1 | SYN Flood 防护 |
| `net.ipv4.conf.all.rp_filter` | 1 | 反向路径过滤 |

| `net.ipv6.conf.all.accept_redirects` | 0 | 禁止 IPv6 ICMP 重定向 |
| `net.ipv4.ip_forward` | 0 | 禁止 IP 转发（非路由器） |

### 3.1 兼容性要求

| 发行版 | 版本 | 支持状态 |
|--------|------|----------|
| CentOS | 7.x | ✅ 支持 |

| AlmaLinux | 8/9 | ✅ 支持 |

### 3.2 性能要求

- 脚本执行时间 < 10 分钟（不含用户交互时间）
- 不影响服务器正常运行
- 不重启服务器（除非必要且用户确认）

### 3.3 可靠性要求

- 每个步骤执行前备份原始配置
- 任何步骤失败时提供回滚选项

- 网络操作有重试机制

### 3.4 用户体验要求

- 彩色输出（绿色=成功，红色=错误，黄色=警告，蓝色=信息）
- 中英文双语支持（默认中文）
- 每个步骤前显示说明
- 操作完成后显示总结报告
- 提供配置文件清单供复查

## 6. 安全考虑

### 6.1 脚本自身安全

| 项目 | 说明 |
|------|------|
| **完整性校验** | 提供 SHA256 校验和 |
| **签名验证** | 可选 GPG 签名 |
| **来源验证** | 从官方仓库下载 |
| **权限检查** | 必须以 root 运行 |

### 6.2 操作安全

| 项目 | 说明 |
|------|------|
| **SSH 断开保护** | 修改 SSH 配置前确保有备用访问方式 |
| **超时保护** | 关键操作有超时机制 |
| **回滚机制** | 每个步骤可独立回滚 |
| **确认机制** | 危险操作需要二次确认 |

### 6.3 SSH 锁定防护

```
⚠️ 重要：在修改 SSH 配置前，脚本必须：
1. 检测当前是否有活跃的 SSH 会话
2. 确保有备用登录方式（控制台/VNC）
3. 配置修改后自动测试新配置
4. 设置定时任务，如果 5 分钟内无新连接自动回滚
```
